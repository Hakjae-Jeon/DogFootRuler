# 6. Security & Guardrails

## 1) Allowlist (필수)
- `ALLOWED_USER_IDS`에 포함된 user_id만 실행.
- 초기 테스트 중 allowlist를 비워둘 수 있으나, 운영 전 반드시 설정.

## 2) 승인 게이트
- 기본 SAFE: 변경/머지 금지
- /apply → 변경 허용
- /merge → 최종 승인(항상 수동)

## 3) 작업 경로 고정
- REPO_ROOT를 고정하고 그 안에서만 실행.
- 임의 경로 접근 금지.

## 4) 금지 행위(초기)
- 시스템 파일/계정 변경
- 네트워크로 비밀 전송(임의 업로드)
- rm -rf 등 파괴적 명령
- 환경변수/토큰 출력

## 5) 비밀 마스킹
- stdout/stderr/summary에서 토큰 패턴을 마스킹 후 텔레그램 전송.
- `.env`, `secrets/` 등 민감 폴더는 Codex 프롬프트/컨텍스트에서 제외 권장.

## 6) 데이터 보관
- runs/는 로컬에 저장.
- 필요 시 7~30일 보관 정책 적용(자동 삭제 옵션).
